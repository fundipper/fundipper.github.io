新建文档

2021-01-01

file document

该指令用于新建文档，在项目目录执行

fungo file document your-document-name

当提示create new file success时，即为执行成功

进入目录

➜  cd your-site-name/content

使用 tree 命令查看目录

➜ tree -L 1
.
└── your-document-name.md

使用 cat 命令查看文件

➜ cat your-document-name.md

---
title: your-document-name
description: ""
keywords: ""
date: "2022-03-30"
lang: ""
slug: ""
---

文章由meta和content两部分构成

默认使用source/article.yaml中内容生成meta信息
用户可自行修改yaml内容，用以实现个性化meta定义

content信息为用户撰写的 markdown

title

文章名称，默认为当前文件名，可自行修改

description

文章描述，可不设置，主要用于SEO

keywords

文章关键词，可不设置，主要用于SEO

date

文章日期，默认为当前日期

设置后可通过/archive/your-archive-date/，访问该归档下所有文章

lang

文章语言，可不设置，主要用于国际化

lang需对应为国际化配置的语言列表

假设文件路径为doc/your-document-name.md

当lang = ""时, 访问地址为:

/doc/your-document-name/

当lang = "zh"时, 访问地址为:

/zh/doc/your-document-name/

若需同时支持多个预言版本时，因文件名不可重复

建议设置为如下模式:

默认语言

/doc/your-document-name.md

中文版本

/doc/your-document-name.zh.md

英文版本

/doc/your-document-name.en.md

slug

文章访问路径，可不设置，主要用于SEO

假设文件路径为doc/your-document-name.md

当slug = ""时, 访问地址为:

/doc/your-document-name/

当slug = "this-is-a-slug-url"时, 访问地址为:

/doc/this-is-a-slug-url/

当slug为空时，则文章URL为文件名

当slug非空时，则文章URL为slug

相同目录下文件的slug不可重复
